# 语音合成功能问题解决方案

## 问题分析

经过检查，发现存在两个主要问题：

1. **语音合成功能不可用**：Coqui TTS和Azure Speech Service无法正常工作
2. **错误信息显示格式问题**：错误信息垂直堆叠显示，可读性差

## 解决方案

### 1. 修复错误信息显示格式

已经修复了错误信息的显示格式问题，通过在`index.html`文件中为错误信息添加了`whitespace-normal break-words`类，确保文本横向排列并正确显示。

### 2. 解决语音合成功能不可用问题

由于环境存在问题，需要重新创建虚拟环境并安装必要的依赖项：

#### 步骤1：删除旧的虚拟环境

```powershell
Remove-Item -Path venv -Recurse -Force
```

#### 步骤2：创建新的虚拟环境

```powershell
python -m venv venv
```

#### 步骤3：激活虚拟环境

```powershell
venv\Scripts\Activate
```

#### 步骤4：升级pip

```powershell
pip install --upgrade pip
```

#### 步骤5：安装必要的依赖项

```powershell
pip install fastapi uvicorn pydantic python-dotenv gTTS pyttsx3 azure-cognitiveservices-speech pydub
```

#### 步骤6：安装Coqui TTS

```powershell
pip install TTS
```

#### 步骤7：配置Azure Speech Service环境变量

在项目根目录创建`.env`文件，添加以下内容：

```
# Azure Speech Service Configuration
AZURE_SPEECH_KEY=your_azure_speech_key_here
AZURE_SPEECH_REGION=your_azure_speech_region_here
```

#### 步骤8：启动服务

```powershell
venv\Scripts\python main.py
```

### 3. 验证解决方案

服务启动后，访问以下地址检查模型可用性：

```
http://localhost:8080/api/speech/models
```

### 4. 故障排除

如果仍然遇到问题，请检查以下几点：

1. 确保Azure Speech Service的API密钥和区域配置正确
2. 确保网络连接正常，能够访问外部服务
3. 检查Python版本是否兼容（推荐使用Python 3.7+）
4. 查看服务启动日志，了解具体的错误信息

### 5. 备用方案

如果Coqui TTS和Azure Speech Service仍然不可用，系统会自动 fallback 到本地的pyttsx3模型，确保基本的语音合成功能能够正常工作。
